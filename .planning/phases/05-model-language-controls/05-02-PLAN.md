# Plan 05-02: UI — переключатель модели и селектор языка

**Phase:** 5 — Model & Language Controls  
**Wave:** 2 (зависит от 05-01)  
**Goal:** Добавить в `HomeScreen` переключатель модели и селектор языка; сохранять выбор между сессиями; передать `TranscriptionOptions` через `ProcessingArgs` в `ProcessingScreen`, который пробрасывает их в контроллеры.

---

## Context

**Зависимости:** 05-01 выполнен — `TranscriptionOptions`, `TranscriptionOptionsRepository`, обновлённые `GroqApiService` и контроллеры доступны.

**Входные данные:** `HomeScreen` с `_onTranscribeTap()` → `ProcessingArgs(file, metadata)`. `ProcessingScreen.initState` создаёт `TranscriptionController` и `ChunkedTranscriptionController`.

**Выходные данные:** Пользователь видит переключатель модели и дропдаун языка прямо на главном экране; выбор сохраняется; запрос к Groq использует выбранную модель и язык.

## Threat Model

| # | Угроза | Митигация |
|---|--------|-----------|
| T-01 | `load()` в `initState` — race condition с первым `build` | `_options` инициализируется дефолтом `TranscriptionOptions.defaults()`; `load()` вызывает `setState` по завершении |
| T-02 | `save()` на каждый тап — лишние I/O | Сохранять только при изменении значения (`_options != newOptions`) |
| T-03 | `ProcessingScreen` читает `args.options` до того как оно установлено | `options` имеет `const` дефолт в `ProcessingArgs` — безопасно |

#### T-01.3 — Загружать настройки в `initState` / `_load`


#### T-01.4 — Callback сохранения при изменении

```dart
Future<void> _onOptionsChanged(TranscriptionOptions updated) async {
  if (updated == _options) return;
  setState(() => _options = updated);
  await _optionsRepo.save(updated);
}
```

#### T-01.6 — Передать `options` в `ProcessingArgs`

В `_onTranscribeTap()` найти:
```dart
arguments: ProcessingArgs(file: _selectedFile!, metadata: _metadata),
```
Заменить на:
```dart
arguments: ProcessingArgs(
  file: _selectedFile!,
  metadata: _metadata,
  options: _options,
),
```

### T-02 — Обновить `lib/ui/screens/processing_screen.dart`

**Файл:** `lib/ui/screens/processing_screen.dart` (MODIFY)

#### T-02.1 — Добавить import

```dart
import '../../features/transcription/transcription_options.dart';
```

#### T-02.2 — Добавить поле `_transcriptionOptions` в `_ProcessingScreenState`

```dart
/// Настройки транскрибации, переданные с HomeScreen через ProcessingArgs.
TranscriptionOptions _transcriptionOptions = const TranscriptionOptions.defaults();
```

#### T-02.3 — Читать `options` из `ProcessingArgs` в `didChangeDependencies()`

> **Важно:** `ProcessingScreen` читает `ModalRoute.of(context)!.settings.arguments` в `didChangeDependencies()`, а **не** в `initState()` (в `initState` маршрут ещё не прикреплён). Найти существующий блок извлечения `args`, где установлен `_file = args.file`, и добавить рядом:

```dart
// В didChangeDependencies(), после _file = args.file:
_transcriptionOptions = args.options;
```

#### T-02.4 — Передать `_transcriptionOptions` в `_startProcessing()`

Найти вызов `_controller.start(_file!)` внутри `_startProcessing()` и обновить:
```dart
_controller.start(_file!, options: _transcriptionOptions);
```

Найти вызов `_chunkedController!.start(normalizedFile)` и обновить:
```dart
_chunkedController!.start(normalizedFile, options: _transcriptionOptions);
```

#### T-02.5 — Обновить retry-кнопку в `_buildChunkedBody()`

Найти кнопку «Повторить» в `_buildChunkedBody()` (вызывает `_chunkedController?.start(_file!)`):
```dart
// Было:
onPressed: () => _chunkedController?.start(_file!),
// Стало:
onPressed: () => _chunkedController?.start(_file!, options: _transcriptionOptions),
```

> `_transcriptionOptions` — поле state, доступно в любом методе `_ProcessingScreenState`. Извлекается один раз в `didChangeDependencies()` и не меняется в течение жизни экрана.
