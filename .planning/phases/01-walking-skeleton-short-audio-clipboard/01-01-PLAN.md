# Plan 01 — Walking Skeleton: Flutter Init + Design System + CI

## Phase Goal (MVP user story)

**As a** студент с записью лекции на телефоне, **I want to** установить APK ezctx и пройти через минимальный навигационный каркас (Home → Settings → API-ключи) с реальным read/write в защищённое хранилище, **so that** последующие планы получают рабочий end-to-end Flutter-проект без архитектурных решений по ходу разработки.

<objective>
Создать с нуля Flutter-проект под Android, поднять CI на GitHub Actions, перенести дизайн-токены из React-прототипа в Dart, реализовать защищённое хранилище ключей и собрать минимальный навигационный каркас из 5 экранов. Этот план — Walking Skeleton: он не реализует ни file_picker, ни Groq, ни Clipboard, но фиксирует архитектуру для всех последующих планов фазы.

Purpose: Закрепить технологические решения (Flutter 3.27.4, minSdk 24, Java 17, ffmpeg_kit_flutter_new), чтобы планы 02–05 не тратили контекст на skeleton-вопросы. Доставить одну реальную end-to-end операцию (запись и чтение в flutter_secure_storage) как доказательство, что выбранный стек работает на устройстве.

Output:
- Чистый Flutter-проект, собирающийся `flutter build apk --debug` локально и в CI.
- Дизайн-система в коде: токены + 5 базовых glass-виджетов.
- 5 экранов-заглушек, связанных навигацией.
- GitHub Actions workflow на push в main с публикацией APK как артефакта.
- Wave 0 тесты (заглушки) для всех будущих unit-suite файлов.
</objective>
