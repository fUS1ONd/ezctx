    - `final double durationSeconds` — длительность нормализованного файла

    Конструктор const с named параметрами.

    ---

    Создать файл `lib/features/transcription/audio_normalization_service.dart`:

    Класс AudioNormalizationService с конструктором:
    ```
    const AudioNormalizationService({
      Future<void> Function(String command)? ffmpegOverride,
      AudioChunkingService? chunkingService,
    });
    ```
    Поля: `_ffmpegOverride` и `_chunkingService` (по умолчанию `const AudioChunkingService()`).

    Метод `Future<NormalizedAudioFile> normalize(String inputPath)`:
    1. Получить tmpDir через `(await getTemporaryDirectory()).path`.
    2. Сформировать outPath: `'$tmpDir/ezctx_norm_${DateTime.now().millisecondsSinceEpoch}.mp3'`
    3. Команда ffmpeg (per D2):
       `'-i "$inputPath" -b:a 32k -ac 1 -ar 16000 -codec:a libmp3lame -y "$outPath"'`
       Пути ОБЯЗАТЕЛЬНО в двойных кавычках внутри строки команды — защита от пробелов в именах файлов (pitfall 2).
    4. Если `_ffmpegOverride != null` — вызвать `await _ffmpegOverride!(command)`.
       Иначе — Completer-паттерн:
         - `final completer = Completer<void>();`
         - `await FFmpegKit.executeAsync(command, (session) async { ... });`
         - `await completer.future;`
       В callback: если `ReturnCode.isSuccess(rc)` — `completer.complete()`,
       иначе — `completer.completeError(const InternalException('ffmpeg: нормализация завершилась с ошибкой'))`.
       НЕ забыть оба await — pitfall 1.
    5. После ffmpeg: `final meta = await _chunkingService.getMetadata(outPath);`
    6. Вернуть `NormalizedAudioFile(path: outPath, durationSeconds: meta.durationSeconds)`.

    Импорты:
    - `dart:async`, `dart:io`
    - `package:ffmpeg_kit_flutter_new/ffmpeg_kit.dart`
    - `package:ffmpeg_kit_flutter_new/return_code.dart`
    - `package:path_provider/path_provider.dart`
    - `package:ezctx/core/error/app_exception.dart`
    - `audio_chunking_service.dart`
    - `normalized_audio_file.dart`

    Добавить `-y` флаг обязателен — pitfall 3 (без него ffmpeg зависнет на retry).

    В `lib/core/constants/app_constants.dart` добавить две константы внутри класса AppConstants:

    ```
    /// Порог для isChunked: длительность нормализованного mp3 > 75 мин = 4500 с (per D3).
    static const int kChunkThresholdSeconds = 4500;

    /// Длительность одного чанка при нарезке нормализованного mp3 (per D3).
    /// 75 мин × 240 КБ/мин ≈ 17.6 МБ < 19 МБ лимита Groq.
    static const double kChunkDurationSeconds = 4500.0;
    ```

    ---

    В `lib/features/transcription/audio_chunking_service.dart`:

    1. Обновить top-level константу `kChunkDurationSeconds` с 1200.0 до 4500.0 (per D3).
       Комментарий изменить: "4500s = 75 мин, ~17.6 МБ при 32k — гарантированно < 19 МБ лимита Groq".

    2. В методе `split()` изменить команду ffmpeg:
       Было:
         `'-i "$filePath" -f segment -segment_time ${kChunkDurationSeconds.toInt()}'`
         `' -c:a libmp3lame -b:a 128k -ac 1 -ar 16000'`
         `' "$tmpBase/chunk_%03d.mp3"'`
       Стало:
         `'-i "$filePath" -f segment -segment_time ${kChunkDurationSeconds.toInt()}'`
         `' -c:a copy'`
         `' "$tmpBase/chunk_%03d.mp3"'`

       Причина: вход split() — уже нормализованный mp3 32k/16kHz/Mono.
       Перекодирование добавляет generation loss и удваивает время нарезки.
       `-c:a copy` для CBR mp3 с -f segment работает корректно (см. RESEARCH.md pitfall 5).

    3. Обновить комментарий над командой: "Команда ffmpeg: сегментация по времени, copy (вход уже mp3 32k)".
