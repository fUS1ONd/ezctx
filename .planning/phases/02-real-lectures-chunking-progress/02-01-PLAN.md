# Plan 02-01: AudioChunkingService — ffprobe-метаданные + ffmpeg-разбивка

## Objective

Создать сервис `AudioChunkingService`, который через `ffmpeg_kit_flutter_new`:
1. Получает метаданные файла (длительность из ffprobe, размер из File.stat).
2. Разбивает аудиофайл на сегменты ≤ 19 МБ (1200s, mp3 128k mono 16kHz) в tmp-директории.

Это фундамент для Plan 02-02 (ChunkedTranscriptionController).

## Context

Phase 1 построила:
- `SelectedAudioFile` — path + name + sizeBytes + extension
- `AppException` hierarchy — `InternalException` для ошибок парсинга/ffmpeg
- `ffmpeg_kit_flutter_new ^4.1.0` уже в pubspec.yaml
- `path_provider ^2.1.5` уже в pubspec.yaml

Важные API ffmpeg_kit_flutter_new:
- `FFprobeKit.getMediaInformation(path)` → `MediaInformationSession`
- Длительность: `session.getMediaInformation()?.getDuration()` — возвращает **String? миллисекунды**, делить на 1000.0
- Запуск ffmpeg: `FFmpegKit.executeAsync(command, completeCallback)` — оборачивать в Completer
- Проверка успеха: `ReturnCode.isSuccess(returnCode)` где returnCode = `session.getReturnCode()`
- Tmp-директория: `path_provider` → `getTemporaryDirectory()`

### Task 1: AudioMetadata + AudioChunkingService

**Goal:** Создать доменную модель `AudioMetadata` и сервис `AudioChunkingService`.

**Files:**
- `lib/features/transcription/audio_metadata.dart` — новый файл, модель метаданных
- `lib/features/transcription/audio_chunking_service.dart` — новый файл, сервис

**Implementation:**

`audio_metadata.dart`:
```
class AudioMetadata {
  final String name;
  final double durationSeconds;  // из ffprobe (ms / 1000.0)
  final int sizeBytes;           // из File.statSync().size

  // durationFormatted: "1:23:45" или "23:45"
  String get durationFormatted { ... }
  String get sizeFormatted { ... } // "123.4 МБ"
}
```

`audio_chunking_service.dart`:

Метод `getMetadata(String filePath) → Future<AudioMetadata>`:
- Вызов `FFprobeKit.getMediaInformation(filePath)`
- `session.getMediaInformation()?.getDuration()` → String? ms → делить на 1000.0
- Если результат null или парсинг не удался → бросить `InternalException('Не удалось получить длительность файла')`
- `File(filePath).statSync().size` для sizeBytes
- `path.basename(filePath)` для name (использовать `package:path` или `dart:io` + split)

Метод `split(String filePath, {String? outputDir}) → Future<List<File>>`:
- tmpDir = `(await getTemporaryDirectory()).path + '/ezctx_chunks_${DateTime.now().millisecondsSinceEpoch}'`
- Создать Directory(tmpDir)
- ffmpeg-команда: `-i "$filePath" -f segment -segment_time 1200 -c:a libmp3lame -b:a 128k -ac 1 -ar 16000 "$tmpDir/chunk_%03d.mp3"`
- Обернуть `FFmpegKit.executeAsync` в Completer<void>:
  ```dart
  final completer = Completer<void>();
  await FFmpegKit.executeAsync(command, (session) async {
    final rc = await session.getReturnCode();
    if (ReturnCode.isSuccess(rc)) {
      completer.complete();
    } else {
      completer.completeError(InternalException('ffmpeg завершился с ошибкой'));
    }
  });
  await completer.future;
  ```
- Собрать список файлов: `Directory(tmpDir).listSync().whereType<File>().toList()..sort(...)`
- Сортировать по имени (chunk_000 < chunk_001 < ...)
- Вернуть отсортированный `List<File>`

### Task 2: Юнит-тесты AudioChunkingService

**Goal:** Покрыть `getMetadata` и `split` юнит-тестами с mock FFmpegKit/FFprobeKit.

**Files:**
- `test/features/transcription/audio_chunking_service_test.dart` — новый файл

**Implementation:**

Поскольку `FFprobeKit` и `FFmpegKit` — статические классы из нативного плагина, тестирование выполнять через инверсию зависимостей. Для этого в `AudioChunkingService` добавить инжектируемые callback-и:

```dart
class AudioChunkingService {
  // Инжектируются только в тестах; по умолчанию — реальные вызовы плагина.
  final Future<MediaInformationSession> Function(String)? _probeOverride;
  final Future<FFmpegSession> Function(String, void Function(FFmpegSession))? _ffmpegOverride;

  const AudioChunkingService({
    Future<MediaInformationSession> Function(String)? probeOverride,
    Future<FFmpegSession> Function(String, void Function(FFmpegSession))? ffmpegOverride,
  }) : _probeOverride = probeOverride,
       _ffmpegOverride = ffmpegOverride;
}
```

Тесты:

1. `getMetadata` — успех: mock возвращает duration "5000" (ms) → ожидаем `durationSeconds == 5.0`
2. `getMetadata` — null duration: mock возвращает null → ожидаем `InternalException`
3. `getMetadata` — непарсируемая строка: mock возвращает "abc" → ожидаем `InternalException`
4. `split` — успех: mock ffmpeg вызывает complete-callback с success-кодом → метод завершается без исключения
5. `split` — ошибка ffmpeg: mock вызывает complete с error-кодом → ожидаем `InternalException`

Использовать `test` package (уже в flutter_test SDK). Все mock-и — ручные (без mockito), через лямбды.
