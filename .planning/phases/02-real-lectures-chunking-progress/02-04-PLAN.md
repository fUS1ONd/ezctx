# Plan 02-04: Интеграция — маршрутизация + ResultScreen + e2e тест

## Objective

Замкнуть end-to-end пайплайн Phase 2:
1. `FileValidator` разрешает файлы > 19 MB (требование TRANS-08 — chunking позволяет большие файлы).
2. `ResultScreen` корректно отображает таймкодированный текст из chunked-режима.
3. Интеграционный тест проверяет весь пайплайн: split → upload → assembly → ChunkedSuccess.

## Context

Из Phase 1:
- `FileValidator` — содержит `maxFileSizeBytes` ограничение, вероятно 19 MB; нужно убрать или поднять
- `ResultScreen` — показывает `r.text` в `SelectableText`, `r.duration` в метаданных; работает без изменений если text уже содержит таймкоды

Из Plan 02-02:
- `ChunkedTranscriptionController` собирает финальный текст в формате `[HH:MM:SS] segment.text\n...`
- `ChunkedSuccess` содержит `TranscriptionResult result` с уже собранным `text`

Из Plan 02-03:
- `ProcessingScreen` направляет на `AppConstants.routeResult` с `ResultArgs(file, result)` — обоих режимов

`ResultScreen` получает `ResultArgs` — он уже не знает, был ли файл chunked. Это правильно.

## Tasks

### Task 1: FileValidator + ResultScreen — финальная полировка

**Goal:** Убрать ограничение размера файла в `FileValidator`, убедиться что `ResultScreen` корректно показывает таймкодированный текст.

**Files:**
- `lib/features/transcription/file_validator.dart` — прочитать и обновить
- `lib/ui/screens/result_screen.dart` — минимальное обновление если нужно

**Implementation:**

`file_validator.dart`:

Прочитать текущую реализацию. Найти константу или проверку размера (что-то вроде `if (sizeBytes > maxFileSizeBytes)`).

Изменить: удалить или закомментировать ограничение максимального размера файла.
Если есть UI-подсказка "до 19 МБ" в file_validator или home_screen — убрать из валидатора
(в HomeScreen tooltip можно оставить как "Phase 1 flow для файлов без chunking").

Важно: валидация расширения (.mp3, .m4a, .wav, .ogg, .flac) и минимального размера (> 0 байт) остаётся.

`result_screen.dart`:

`SelectableText(r.text)` уже рендерит любой текст. Таймкоды вида `[00:00:00] текст\n` отобразятся корректно как обычный текст.

Проверить строку метаданных: `'${file.sizeFormatted} · ${file.extension.toUpperCase()} · ${r.duration.toStringAsFixed(1)} сек'`
— для chunked-результата `r.duration` будет суммой длительностей всех чанков. Изменить формат:
если `r.duration >= 60` — показывать `AudioMetadata`-style формат через форматирование секунд:
```dart
String _formatDuration(double seconds) {
  final total = seconds.toInt();
  final h = total ~/ 3600;
  final m = (total % 3600) ~/ 60;
  final s = total % 60;
  if (h > 0) return '${h}ч ${m}мин';
  if (m > 0) return '${m}мин ${s}с';
  return '${s}с';
}
```

Кнопка "Скопировать" — без изменений: `Clipboard.setData(ClipboardData(text: r.text))` копирует полный текст с таймкодами.

### Task 2: Интеграционный тест chunked flow

**Goal:** Тест, воспроизводящий полный пайплайн Phase 2 с моками.

**Files:**
- `test/features/transcription/integration_chunked_flow_test.dart` — новый файл

**Implementation:**

Тест использует mockito-моки для `GroqApiService`, `AudioChunkingService`, `ApiKeyRepository`.

Сценарий 1 — happy path, 2 чанка:
```
1. mock AudioChunkingService.split() → [File('chunk_000.mp3'), File('chunk_001.mp3')]
2. mock GroqApiService.transcribeChunk() для chunk 0 → TranscriptionResult(
     text: 'Привет мир', segments: [TranscriptionSegment(start: 0.0, end: 2.0, text: 'Привет мир')]
   )
3. mock GroqApiService.transcribeChunk() для chunk 1 → TranscriptionResult(
     text: 'Как дела', segments: [TranscriptionSegment(start: 0.0, end: 1.5, text: 'Как дела')]
   )
4. ChunkedTranscriptionController.start(file)
5. Ожидаем ChunkedSuccess
6. result.text содержит '[00:00:00] Привет мир' и '[00:20:00] Как дела'
   (chunkDuration = 1200s, chunk 1 offset = 1200s = 20 мин)
```

Сценарий 2 — retry и успех:
```
1. mock split → 1 чанк
2. mock transcribeChunk вызов 1: бросает NetworkException
3. mock transcribeChunk вызов 2: возвращает успех
4. Итоговое состояние: ChunkedSuccess (не ChunkedError)
5. Проверить что mock вызван 2 раза
```

Сценарий 3 — исчерпаны retries:
```
1. mock split → 1 чанк  
2. mock transcribeChunk: всегда бросает NetworkException
3. После 3 попыток → ChunkedError
```

Сценарий 4 — cleanup всегда выполняется:
```
1. mock split → 2 файла (File-объекты с mock delete)
2. mock transcribeChunk для второго чанка бросает AuthException
3. Конечное состояние ChunkedError
4. Оба файла получили delete() вызов
```

Использовать `TestWidgetsFlutterBinding.ensureInitialized()` только если нужно.
Предпочесть чистый `dart:test` без Flutter binding для скорости.
