# Plan 02-03: UI — карточка метаданных + chunked-экран прогресса

## Objective

Расширить `HomeScreen` для показа метаданных файла (ffprobe-длительность) и обновить
`ProcessingScreen` для chunked-режима — общий прогресс-бар и список `ChunkTile`.
Зарегистрировать `ChunkedTranscriptionController` в дереве виджетов.

## Context

Из Phase 1:
- `HomeScreen` — StatefulWidget, уже показывает `_buildFilePreview(file)` с именем и размером
- `ProcessingScreen` — принимает `SelectedAudioFile` через `ModalRoute.of(context)?.settings.arguments`
- `GlassCard`, `GlassTile`, `GlassIconBtn`, `PrimaryButton`, `ShimmerBar` — готовые виджеты
- `AppTextStyles`, `AppColors`, `AppSpacing`, `AppGradients` — design tokens из `design_tokens.dart`
- `AppConstants.routeProcessing`, `AppConstants.routeResult` — маршруты навигации

Из Plan 02-01 (создан ранее):
- `AudioChunkingService.getMetadata(path)` → `Future<AudioMetadata>`
- `AudioMetadata` — `name`, `durationSeconds`, `sizeBytes`, `durationFormatted`, `sizeFormatted`

Из Plan 02-02 (создан ранее):
- `ChunkedTranscriptionController` — ChangeNotifier, метод `start(SelectedAudioFile)`
- `ChunkedState` sealed class: `ChunkedSplitting`, `ChunkedProcessing(chunks, completedCount, totalCount, progress)`, `ChunkedSuccess`, `ChunkedError`, `ChunkedMissingKey`
- `ChunkState` sealed class: `ChunkWaiting`, `ChunkUploading`, `ChunkDone`, `ChunkRetrying`, `ChunkFailed`

Маршрутизация: файл < 19 MB → `TranscriptionController` (Phase 1 путь), файл ≥ 19 MB → `ChunkedTranscriptionController`. Порог: `AppConstants.maxFileSizeBytes` (уже определён в Phase 1) или константа 19 * 1024 * 1024.

**ВАЖНО:** Не вводить новые pubspec-зависимости. Для Provider использовать то, что уже есть.
Если `provider` не в pubspec — использовать `InheritedNotifier` или создать
`ChunkedTranscriptionController` прямо в `ProcessingScreen` (как это сделано с `TranscriptionController`).

## Tasks

### Task 1: HomeScreen — загрузка метаданных + карточка

**Goal:** После pick файла асинхронно загрузить метаданные через ffprobe и показать обновлённую карточку с длительностью.

**Files:**
- `lib/ui/screens/home_screen.dart` — расширить `_HomeScreenState`

**Implementation:**

Добавить в `_HomeScreenState`:
```dart
AudioMetadata? _metadata;
bool _loadingMetadata = false;
```

Метод `_loadMetadata(SelectedAudioFile file)`:
```dart
Future<void> _loadMetadata(SelectedAudioFile file) async {
  setState(() { _loadingMetadata = true; });
  try {
    final meta = await AudioChunkingService().getMetadata(file.path);
    if (mounted) setState(() { _metadata = meta; });
  } catch (_) {
    // Тихо: если ffprobe не удался, метаданные показываем без длительности.
  } finally {
    if (mounted) setState(() { _loadingMetadata = false; });
  }
}
```

Вызывать `_loadMetadata(f)` после `setState(() => _selectedFile = f)` в `_onUploadTap()`.

Обновить `_buildFilePreview` — добавить строку с длительностью:
- Если `_loadingMetadata == true` → показать маленький `CircularProgressIndicator(strokeWidth: 1.5)` размером 12×12 рядом с size
- Если `_metadata != null` → показать `'${_metadata!.sizeFormatted} · ${_metadata!.durationFormatted}'`
- Если `_metadata == null && !_loadingMetadata` → показать только `'${file.sizeFormatted} · ${file.extension.toUpperCase()}'` (как было)

Сбросить `_metadata = null` и `_loadingMetadata = false` при повторном выборе файла.

Передать `_metadata` в аргументы маршрута при навигации на `/processing` — упаковать в новый класс `ProcessingArgs`:
```dart
class ProcessingArgs {
  final SelectedAudioFile file;
  final AudioMetadata? metadata; // null для коротких файлов без ffprobe
  const ProcessingArgs({required this.file, this.metadata});
}
```
Создать файл `lib/features/transcription/processing_args.dart`.

В `_onTranscribeTap()` заменить `arguments: _selectedFile` на `arguments: ProcessingArgs(file: _selectedFile!, metadata: _metadata)`.

### Task 2: ChunkTile + ProcessingScreen chunked-режим + регистрация контроллера

**Goal:** Создать виджет `ChunkTile`, расширить `ProcessingScreen` для chunked-режима, зарегистрировать `ChunkedTranscriptionController`.

**Files:**
- `lib/ui/widgets/chunk_tile.dart` — новый файл
- `lib/ui/screens/processing_screen.dart` — расширить для chunked-режима
- `lib/main.dart` — зарегистрировать `ChunkedTranscriptionController`
- `lib/features/transcription/processing_args.dart` — новый файл (создаётся здесь или в Task 1)

**Implementation:**

`chunk_tile.dart` — виджет одного чанка:
```dart
class ChunkTile extends StatelessWidget {
  final ChunkState state;
  const ChunkTile({super.key, required this.state});

  @override
  Widget build(BuildContext context) {
    // Цвет и иконка по типу ChunkState:
    // ChunkWaiting → AppColors.inkTertiary, Icons.hourglass_empty
    // ChunkUploading → AppColors.accent, CircularProgressIndicator (маленький, strokeWidth 2)
    // ChunkDone → AppColors.good, Icons.check_circle_outline
    // ChunkRetrying(attempt) → AppColors.warn (AppColors.accent как fallback), Icons.refresh
    // ChunkFailed → AppColors.bad, Icons.error_outline
    //
    // Показывать: иконка + "Чанк N" + подпись статуса (Ожидает / Отправляется / Готов / Повтор N / Ошибка)
    // Завернуть в GlassTile или простой Container с AppSpacing.sm padding
  }
}
```

`processing_screen.dart` — расширить для поддержки обоих режимов:

В `didChangeDependencies()` изменить логику аргументов:
```dart
final args = ModalRoute.of(context)?.settings.arguments;
if (args is ProcessingArgs) {
  _file = args.file;
  _isChunked = args.file.sizeBytes >= 19 * 1024 * 1024;
  // ...
} else if (args is SelectedAudioFile) {
  // Обратная совместимость — старый путь
  _file = args;
  _isChunked = false;
}
```

Добавить поле `bool _isChunked = false` в `_ProcessingScreenState`.

Добавить `ChunkedTranscriptionController? _chunkedController` в State.
Если `_isChunked == true` → инициализировать `_chunkedController` и вызвать `_chunkedController!.start(_file!)`.
Если `_isChunked == false` → продолжить использовать существующий `_controller` (TranscriptionController).

В `build()` добавить branch:
```dart
if (_isChunked && _chunkedController != null) {
  return _buildChunkedBody(_chunkedController!.state);
} else {
  // существующий виджет-дерево для single-file режима
}
```

`_buildChunkedBody(ChunkedState state)`:
- Если `ChunkedSplitting` → показать `ShimmerBar` + текст "Разбиваем на чанки..."
- Если `ChunkedProcessing(chunks, progress)`:
  - LinearProgressIndicator(value: state.progress) под карточкой файла
  - Текст "N из M чанков готово"
  - `ListView` или `Column` из `ChunkTile` по `state.chunks`
- Если `ChunkedSuccess` → навигация на ResultScreen (аналогично TranscriptionSuccess)
- Если `ChunkedError` → показать сообщение + кнопку "Повторить"
- Если `ChunkedMissingKey` → кнопка "Перейти в настройки"

Регистрация `ChunkedTranscriptionController` в `main.dart`:
Поскольку `provider` отсутствует в pubspec — контроллер создаётся прямо в `_ProcessingScreenState.initState()` (как `TranscriptionController`). Никаких Provider-изменений в `main.dart` не нужно. Убрать упоминание Provider из requires.

Обновить `dispose()` в `_ProcessingScreenState`:
```dart
_chunkedController?.removeListener(_onChunkedStateChange);
_chunkedController?.dispose();
```
