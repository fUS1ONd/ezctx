## Архитектура GroqKeyPool

| Вариант | Описание | Выбран |
|---------|----------|--------|
| ChangeNotifier-синглтон | Push-обновления, ребилд только при изменении статуса, Flutter-идиом | ✓ |
| Чистый сервис + Timer | Поллинг 1 раз/сек, проще, но лишние ребилды и задержка 1с | |

**Выбор:** ChangeNotifier-синглтон
**Пояснение:** Пользователь уточнил вопрос производительности. ChangeNotifier эффективнее: нет лишних ребилдов, мгновенное обновление при смене статуса.

---

## Ротация при 429 (кто управляет)

| Вариант | Описание | Выбран |
|---------|----------|--------|
| Пул blackbox | Контроллер вызывает `reportRateLimited` + `acquireKey`, пул сам решает следующий ключ | ✓ |
| Контроллер управляет вручную | Ловит RateLimitException, блокирует ключ, выбирает следующий | |

**Выбор:** Пул blackbox
**Пояснение:** Чище разделение ответственности — контроллер управляет чанками, пул управляет ключами.

| Вариант | Описание | Выбран |

## Поведение при all-blocked

| Вариант | Описание | Выбран |
|---------|----------|--------|
| Ждёт (авторазблокировка) | pool.acquireKey() засыпает до разблокировки ближайшего ключа | ✓ |
| Падает с ошибкой | ChunkedError: "Все ключи заблокированы", пользователь жмёт Retry вручную | |

**Выбор:** Ждёт
**Пояснение:** Соответствует Success Criteria 5 — часовая лекция расшифровывается без вмешательства пользователя.
