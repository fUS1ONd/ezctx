# Phase 05.5 Plan 01: Clipboard Engine (super_clipboard) Summary

**One-liner:** Замена Clipboard.setData на super_clipboard через ClipboardService с ContentProvider в манифесте — устраняет бесшумное обрезание транскрипций >1 MB на Android.

## What Was Built

Внедрён пакет `super_clipboard` для обхода Android Binder IPC лимита (~1 MB). При стандартном `Clipboard.setData` транскрипции длиннее ~21 минуты обрезались без ошибки. `super_clipboard` регистрирует Android ContentProvider, который передаёт данные напрямую приложению-получателю, минуя Binder.

### Компоненты

1. **pubspec.yaml** — добавлена зависимость `super_clipboard: ^0.9.1`

2. **AndroidManifest.xml** — ContentProvider `com.superlist.super_native_extensions.DataProvider` с authority `com.ezctx.app.SuperClipboardDataProvider`, `android:exported="true"`, `android:grantUriPermissions="true"`

3. **ClipboardService** (`lib/core/services/clipboard_service.dart`) — static final класс с методом `copyText(String text)`:
   - Использует `ClipboardWriter` (абстракция) через `clipboardOverride ?? SystemClipboard.instance`
   - `StateError` при недоступном Clipboard API
   - `@visibleForTesting` поля для изоляции в тестах

4. **result_screen.dart** — заменён `Clipboard.setData` на `ClipboardService.copyText` в `_onCopyTap`, обёрнут в try/catch: при ошибке — SnackBar «Ошибка копирования: ...», кнопка «Скопировано» — только при успехе

5. **Тесты** (`test/core/services/clipboard_service_test.dart`):
   - CP-02a: `write` вызывается с `DataWriterItem` — PASS
   - CP-02b: StateError при `simulateUnavailable=true` — PASS
   - CP-02c: PlatformException из `write` пробрасывается — PASS

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] CP-02b: SystemClipboard.instance не null в тестовой среде**
- **Found during:** Task 2 — первый запуск тестов
- **Issue:** В тестовой среде `SystemClipboard.instance` не возвращает null — при обращении к нативному каналу бросает `NoSuchChannelException` вместо ожидаемого `StateError`
- **Fix:** Добавлен `@visibleForTesting static bool simulateUnavailable = false` в `ClipboardService`. В CP-02b устанавливается `simulateUnavailable = true`, что напрямую проверяет ветку null-guard без обращения к нативному слою
- **Files modified:** `lib/core/services/clipboard_service.dart`, `test/core/services/clipboard_service_test.dart`

**2. [Rule 1 - Bug] clipboardOverride тип изменён с SystemClipboard? на ClipboardWriter?**
- **Found during:** Task 2 — планирование мокирования
- **Issue:** `SystemClipboard` — конкретный класс с приватным конструктором `SystemClipboard._()`, mockito не может его мокировать
- **Fix:** `clipboardOverride` объявлен как `ClipboardWriter?` (абстрактный класс), что позволяет `@GenerateMocks([ClipboardWriter])` генерировать мок
