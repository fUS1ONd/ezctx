# Phase 5.5: Clipboard Engine (super_clipboard) — Context

**Gathered:** 2026-05-18
**Status:** Ready for planning
**Source:** User report + bug analysis

<domain>
## Phase Boundary

Эта фаза исправляет один конкретный баг: кнопка «Скопировать» обрезает транскрипцию длинных аудиофайлов (1.5+ часа) при копировании в буфер обмена на Android. Причина — системный лимит `Clipboard.setData` на Android (ограничение на объём данных в ClipboardManager). Решение — заменить стандартный Flutter Clipboard API на пакет `super_clipboard`, который обходит это ограничение.

Фаза не меняет UI, логику транскрибации, форматы вывода или другие части приложения. Только механизм записи в буфер обмена.

</domain>

<decisions>
## Implementation Decisions

### Проблема (зафиксировано пользователем)
- Транскрипция 1.5-часового аудио обрезается при копировании — в буфере оказывается только текст до ~21-й минуты.
- Стандартный `Clipboard.setData` из Flutter `services` ограничен на Android по объёму.
- Пользователь нашёл пакет `super_clipboard` как потенциальное решение.

### Пакет для использования
- **LOCKED:** Использовать пакет `super_clipboard` для записи в буфер обмена.
- Исследовать совместимость с Flutter 3.x и Android minSdkVersion проекта.
- Исследовать API пакета: как записать plain text, как обработать ошибки.

### Область изменений
- **LOCKED:** Заменить все вызовы `Clipboard.setData` и `Clipboard.getData` на `super_clipboard` API.
- Не трогать UI-компоненты (кнопки, снекбары) — только логику под ними.
- Снек-бар «Скопировано» должен появляться только при успехе; при ошибке — показать сообщение.

### Claude's Discretion
- Конкретный API `super_clipboard` для plain text (требует исследования).
- Нужна ли инициализация пакета в `main.dart` или он работает без неё.
- Как обработать случай, когда `super_clipboard` недоступен на устройстве (fallback или error).
- Нужны ли изменения в `AndroidManifest.xml` или `build.gradle`.

</decisions>

<specifics>
## Specific Ideas

- Пользователь конкретно назвал пакет `super_clipboard` как кандидата для исправления.
- Проблема воспроизводится на реальном устройстве Android с транскрипцией 1.5-часового аудио.
- Текст обрезается примерно на 21-й минуте при plain text режиме.
- Возможно, `super_clipboard` использует Content Provider или другой механизм Android для обхода лимита.
