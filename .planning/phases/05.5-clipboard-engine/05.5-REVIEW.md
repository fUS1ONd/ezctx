**Reviewed:** 2026-05-18
**Depth:** standard

## Summary

Реализация ClipboardService через `super_clipboard` корректна по структуре: обёртка над `ClipboardWriter`, механизм переопределения через `clipboardOverride`/`simulateUnavailable`, обработка ошибок в `_onCopyTap`. Однако обнаружены два критических дефекта: один тест CP-02c содержит структурный изъян и не проверяет то, что заявляет, а ContentProvider в AndroidManifest.xml открыт без ограничений разрешений. Также выявлены два предупреждения: потенциальный RangeError в `_saveTranscriptTxt` и скрытый тихий `catch (_)`.

## Critical Issues

### CR-01: Тест CP-02c не проверяет async-исключение (vacuous pass)

**File:** `test/core/services/clipboard_service_test.dart:65-69`

**Issue:** В тесте CP-02c используется `expect(...)` без `await` для функции, возвращающей `Future`. `ClipboardService.copyText` — async-функция; `expect()` завершается синхронно до разрешения будущего, поэтому бросок `PlatformException` происходит в отдельном микрозадании, которое тест уже не ловит. Тест проходит вне зависимости от реального поведения — тест провальный.

```dart
// Текущий код — тест проходит vacuously:
expect(
  () => ClipboardService.copyText('текст'),
  throwsA(isA<PlatformException>()),
);
```

**Fix:** Заменить на `expectLater` с `await`:

```dart
await expectLater(
  () => ClipboardService.copyText('текст'),
  throwsA(isA<PlatformException>()),
);
```

Это идентично тому, как написан CP-02b в том же файле (строка 48-55) — нужно привести CP-02c к тому же образцу.

### CR-02: ContentProvider открыт без ограничения разрешений — возможен несанкционированный доступ

**File:** `android/app/src/main/AndroidManifest.xml:40-45`

**Issue:** `DataProvider` объявлен с `android:exported="true"` и `android:grantUriPermissions="true"`, но без атрибута `android:readPermission` и без `<permission>` тега. Это означает, что любое приложение на устройстве может обращаться к этому ContentProvider по URI `com.ezctx.app.SuperClipboardDataProvider` и читать данные, предоставляемые провайдером. В контексте приложения, которое хранит транскрипции лекций (потенциально конфиденциальный контент), это раскрывает пользовательские данные другим приложениям без явного согласия.

```xml
<!-- Текущий — без readPermission: -->
<provider
    android:name="com.superlist.super_native_extensions.DataProvider"
    android:authorities="com.ezctx.app.SuperClipboardDataProvider"
    android:exported="true"
    android:grantUriPermissions="true" >
</provider>
```

**Fix:** Добавить `android:readPermission` с permission signature-level, чтобы ограничить доступ только своим приложением. Либо проверить документацию `super_clipboard` на предмет рекомендуемой конфигурации — если плагин требует `exported=true` для системного клипбоарда, то следует явно задокументировать и добавить `android:readPermission="android.permission.CLIPBOARD_SERVICE"` или собственное разрешение уровня `signature`.

Минимальный безопасный вариант (если плагин это поддерживает):

```xml
<provider
    android:name="com.superlist.super_native_extensions.DataProvider"
    android:authorities="com.ezctx.app.SuperClipboardDataProvider"
    android:exported="true"
    android:grantUriPermissions="true"
    android:readPermission="android.permission.BIND_NOTIFICATION_LISTENER_SERVICE" >
</provider>
```

Либо, если плагин не требует доступа от внешних приложений, рассмотреть `android:exported="false"` — но это, вероятно, сломает интеграцию с системным клипбоардом. Рекомендуется свериться с upstream-документацией `super_native_extensions`.

## Warnings

### WR-01: `substring` с `lastIndexOf` может бросить RangeError при пути без `/`

**File:** `lib/ui/screens/result_screen.dart:109-112`

**Issue:** `paths.plainPath.lastIndexOf('/')` возвращает `-1`, если в пути нет символа `/` (например, возвращён просто filename без директории). Вызов `substring(0, -1)` в Dart бросает `RangeError`. Хотя `_saveTranscriptTxt` обёрнута в `try/catch`, ошибка всё равно тихо поглощается, `_savedPath` не устанавливается, и пользователь не видит ни пути, ни ошибки.

```dart
final folderPath = paths.plainPath.substring(
  0,
  paths.plainPath.lastIndexOf('/'), // возвращает -1 если '/' нет
);
```

**Fix:** Добавить защитную проверку:

```dart
final lastSep = paths.plainPath.lastIndexOf('/');
final folderPath = lastSep > 0
    ? paths.plainPath.substring(0, lastSep)
    : paths.plainPath;
setState(() => _savedPath = folderPath);
```

### WR-02: Тихий `catch (_)` в `_saveTranscriptTxt` скрывает любые ошибки

**File:** `lib/ui/screens/result_screen.dart:115-117`

**Issue:** `catch (_) { }` поглощает все исключения без разбора, включая `Error`-подтипы (например, `OutOfMemoryError`, `RangeError` из WR-01 выше). Это нарушает observability: пользователь не узнает, что файл не сохранён, а разработчик не получит сигнала об ошибке. По крайней мере в debug-режиме ошибку следует логировать.

```dart
} catch (_) {
  // тихо — UI не блокируется ошибкой записи
}
```

**Fix:** Добавить условный лог или ограничить перехват до ожидаемых типов:

```dart
} catch (e, st) {
  // Не блокируем UI, но логируем для отладки.
  assert(() {
    debugPrint('[ResultScreen] ошибка сохранения файла: $e\n$st');
    return true;
  }());
}
```

Либо заменить на `on Exception` чтобы не перехватывать `Error`-подтипы, которые сигнализируют о программных ошибках.

**Fix:** Импорт явно обоснован использованием `PlatformException` — при желании оставить как есть либо добавить комментарий `// PlatformException`.
