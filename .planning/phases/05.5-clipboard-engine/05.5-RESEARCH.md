## Summary

Стандартный `Clipboard.setData` Flutter использует Android Binder IPC для передачи данных между процессами. Binder имеет жёсткий лимит транзакции ~1 MB, и при больших текстах (транскрипция 1.5 ч ≈ несколько сотен КБ plain text) Android **бесшумно обрезает** данные, не бросая исключение. Результат — в буфере оказывается только часть текста (~21 минута по наблюдению пользователя).

`super_clipboard` (пакет `superlistapp/super_native_extensions`) обходит это ограничение за счёт другого механизма: вместо прямой передачи данных через Binder пакет регистрирует **Android Content Provider** (`DataProvider`), который отдаёт данные приложению-получателю напрямую, минуя Binder-ограничение на размер транзакции. Это стандартный Android-паттерн для больших объёмов данных в буфере обмена.

Пакет находится в активной разработке (последний релиз 0.9.1 от 2025-06-11), хорошо задокументирован, используется в production. Внутри написан на Rust; для сборки не требуется установка Rust — загружаются precompiled binaries. NDK автоматически загружается при первой сборке (~1 GB, только один раз).

**Primary recommendation:** Заменить `Clipboard.setData` на `super_clipboard` 0.9.1. Единственное обязательное изменение конфигурации — добавить `<provider>` в `AndroidManifest.xml`. Изменений в `build.gradle` и `main.dart` не требуется, так как `minSdk = 24` уже выше требуемого минимума 23.

### Pattern 1: Writing Plain Text

**What:** Минимальный вызов для записи plain text через `super_clipboard`.

**When to use:** Всегда при замене `Clipboard.setData`.

```dart
// Source: https://raw.githubusercontent.com/superlistapp/super_native_extensions/main/super_clipboard/README.md
import 'package:super_clipboard/super_clipboard.dart';

Future<void> copyText(String text) async {
  final clipboard = SystemClipboard.instance;
  if (clipboard == null) {
    // Clipboard API недоступен на данной платформе
    throw Exception('Clipboard not supported');
  }
  final item = DataWriterItem();
  item.add(Formats.plainText(text));
  await clipboard.write([item]);
}
```

### Pattern 2: Error-safe copy с fallback

**What:** Полная обёртка с try/catch для UI.

```dart
// Паттерн для result_screen.dart
Future<void> _onCopyTap() async {
  if (_args == null) return;
  try {
    await ClipboardService.copyText(_currentText);
    if (!mounted) return;
    setState(() => _copied = true);
    Future.delayed(const Duration(milliseconds: 1500), () {
      if (mounted) setState(() => _copied = false);
    });
  } catch (e) {
    if (!mounted) return;
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(content: Text('Ошибка копирования: $e')),
    );
  }
}
```

### Anti-Patterns to Avoid

- **Ленивый callback для больших текстов:** `item.add(Formats.plainText.lazy(() => heavyComputation()))` — main thread может быть заблокирован. Для больших готовых строк передавай текст напрямую, не через lazy.
- **Игнорирование null-проверки `SystemClipboard.instance`:** На некоторых платформах (web, старые Android) API может вернуть null. Без проверки — NPE.
- **Прямое использование `SystemClipboard` в виджете:** Инкапсулируй в сервис, чтобы тест мог мокать.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Обход Android Binder-лимита | Самописный ContentProvider для clipboard | `super_clipboard` | ContentProvider регистрация, intent-filter, lifecycle — сотни строк Kotlin |
| Lazy clipboard data provision | Самописный callback-механизм | `DataWriterItem.addVirtualFile` / `.lazy()` | Нативная реализация на каждой платформе |

**Key insight:** Android ContentProvider для clipboard — не тривиальный код. `super_clipboard` берёт на себя регистрацию, авторизацию (`android:grantUriPermissions="true"`), lifecycle и кросс-платформенную абстракцию.

## Common Pitfalls

### Pitfall 1: ContentProvider не объявлен в AndroidManifest
**What goes wrong:** `super_clipboard` запустится, `SystemClipboard.instance` вернёт не-null, но при попытке write больших данных (изображения, кастомные форматы) появится исключение. Для **plain text** ContentProvider **может быть не обязателен** (текст передаётся напрямую через Binder для малых объёмов), но для обхода Binder-лимита на больших текстах ContentProvider **нужен**.
**Why it happens:** Пакет не может автоматически прописать provider в манифест (это build-time конфигурация).
**How to avoid:** Добавить `<provider>` в `AndroidManifest.xml` как указано в README (см. раздел Code Examples).
**Warning signs:** Копирование работает на коротких текстах, обрезает длинные — скорее всего ContentProvider не зарегистрирован.

### Pitfall 2: Неправильный `android:authorities`
**What goes wrong:** ContentProvider зарегистрирован, но с неверным authority — при обращении к нему Android выбрасывает исключение.
**Why it happens:** Authority должен быть `<ваш-package-name>.SuperClipboardDataProvider`. Package name проекта: `com.ezctx.app`.
**How to avoid:** `android:authorities="com.ezctx.app.SuperClipboardDataProvider"`

### Pitfall 3: Долгий первый билд (NDK ~1 GB)
**What goes wrong:** Разработчик запускает сборку, ждёт 10–30 минут — выглядит как зависание.
**Why it happens:** NDK скачивается автоматически при первой сборке.
**How to avoid:** Предупредить команду, не прерывать сборку.

### Pitfall 4: Rust не установлен — precompiled binaries не найдены
**What goes wrong:** В offline-среде или при сбое CDN сборка падает.
**Why it happens:** По умолчанию пакет скачивает precompiled binaries; если это невозможно — нужен Rust.
**How to avoid:** Для CI/CD убедиться в доступе к `static.crates.io` или заранее установить Rust через `rustup`.

## Code Examples

### AndroidManifest.xml — добавить provider

```xml
<!-- Source: https://github.com/superlistapp/super_native_extensions/tree/main/super_clipboard README -->
<!-- Добавить внутрь <application> в android/app/src/main/AndroidManifest.xml -->
<provider
    android:name="com.superlist.super_native_extensions.DataProvider"
    android:authorities="com.ezctx.app.SuperClipboardDataProvider"
    android:exported="true"
    android:grantUriPermissions="true" >
</provider>
```

### ClipboardService — новый файл

```dart
// lib/core/services/clipboard_service.dart
import 'package:super_clipboard/super_clipboard.dart';

/// Сервис записи в буфер обмена через super_clipboard.
/// Обходит Android Binder-лимит (~1 MB) для длинных транскрипций.
class ClipboardService {
  const ClipboardService._();

  /// Записывает [text] в буфер обмена.
  /// Выбрасывает [StateError] если API недоступен,
  /// или [PlatformException] при ошибке платформы.
  static Future<void> copyText(String text) async {
    final clipboard = SystemClipboard.instance;
    if (clipboard == null) {
      throw StateError('Clipboard API недоступен на этой платформе');
    }
    final item = DataWriterItem();
    item.add(Formats.plainText(text));
    await clipboard.write([item]);
  }
}
```

### result_screen.dart — замена вызова

```dart
// Было:
await Clipboard.setData(ClipboardData(text: _currentText));

// Стало:
await ClipboardService.copyText(_currentText);
// (импорт flutter/services.dart можно убрать если больше нигде не используется)
```
